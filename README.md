## 🛠️ 기술 스택 (Tech Stack)

이 프로젝트는 안정적인 서비스 제공과 효율적인 데이터 처리를 위해 다음과 같은 기술 스택으로 구성되었습니다.

### ☁️ 인프라 및 배포 (Infrastructure & Deployment)
| 구분 | 기술 스택 | 설명 |
| :--- | :--- | :--- |
| **클라우드** | **Amazon EC2 (AWS)** | 서버를 호스팅하고 운영하기 위한 클라우드 컴퓨팅 서비스 |
| **운영체제** | **Ubuntu** | EC2 인스턴스에 설치된 리눅스 운영체제 |
| **가상화** | **Docker** | 프론트엔드와 백엔드 환경을 컨테이너로 분리 및 배포 |

### 💻 애플리케이션 (Application)

| 구분 | 역할 | 기술 스택 |
| :--- | :--- | :--- |
| **프론트엔드 (Frontend)** | 사용자 인터페이스 구축 | **React**, **Node.js** |
| **백엔드 (Backend)** | 핵심 비즈니스 로직 및 API 서버 | **FastAPI** (Python) |


### 💾 데이터베이스 및 데이터 처리 (Database & Data Processing)

#### 데이터베이스
| 구분 | 기술 스택 | 설명 |
| :--- | :--- | :--- |
| **관계형 DB** | **MariaDB** | 정형화된 데이터 저장 및 관리 |
| **벡터 DB** | **Drant** | AI/검색 기능에 필요한 벡터 데이터의 효율적인 저장 및 검색 |

#### 데이터 수집 도구
| 기술 스택 | 역할 |
| :--- | :--- |
| **BeautifulSoup** | HTML/XML 파싱 및 정적 웹 크롤링 |
| **Playwright** | 동적 웹사이트 크롤링 및 자동화 |
| **pandas** | 수집된 데이터의 가공 및 분석 |


### 🌐 외부 연동 API (External APIs)
시스템의 기능을 확장하고 다양한 정보를 제공하기 위해 아래의 외부 API들을 연동하였습니다.

* **OpenAI:** AI 기반의 자연어 처리 및 콘텐츠 생성 기능
* **NAVER:** 검색, 지도, 번역 등 네이버 서비스 연동
* **Google:** 지도, 검색 등 구글 서비스 연동
* **ODSay:** 대중교통 경로 탐색 및 길 찾기 기능

## 💭 프로젝트 회고 (Retrospective)

<details>
<summary><b>1. 🚀 성공 요인</b></summary>
<br>

* **사용자 중심의 명확한 목표 설정:** 외국인 대상 K-여행 플래너 서비스라는 목표 아래, **사용자 편의성과 직관적 사용성(UX)**을 최우선으로 설계하여 프로젝트의 방향성을 명확히 했습니다.
* **전 영역 상호 학습 기반의 협업:** 팀원 모두 프론트엔드, 백엔드, 데이터베이스 관리의 전 영역을 직접 경험하며 상호 학습했습니다. 이를 통해 **개발 흐름 전체에 대한 이해도**를 높이고 성공적인 웹서비스 배포까지 완수할 수 있었습니다.
* **지속적인 기술 검토와 적극적인 문제 해결:** 초기 설계의 부담을 버리고, 발생하는 수정 사항에 두려워하지 않으며 꾸준히 기술적인 검토를 진행하여 프로젝트의 **기능 완성도**를 높였습니다.

</details>

<details>
<summary><b>2. ⛰️ 어려웠던 점 및 극복 과정</b></summary>
<br>

* **기술적인 난관 및 로직 구현의 복잡성 해결:**
    * **어려웠던 점:** 데이터 크롤링, 벡터 DB 연동, 다양한 외부 API 통합 등 기술적 난관과 복잡한 로직 구현 과정에서 어려움을 겪었습니다.
    * **극복 과정:** 대화를 통해 서로 공감하고, 복잡한 로직을 **종이에 직접 그리며 시각화**하고 **틀을 잡는** 방식으로 문제를 단계별로 해체하여 해결했습니다. 또한, 강사님 및 멘토님과의 자문을 적극적으로 활용했습니다.
* **팀원 간의 의견 충돌 및 조율 과정:**
    * **어려웠던 점:** 팀원마다 지식과 관점이 달라 서비스 방향성에 대한 의견 충돌이 발생했습니다.
    * **극복 과정:** 서로의 이해를 돕기 위해 **그림을 그려 구조를 시각화**하고, 끊임없는 대화를 통해 이해되지 않는 부분을 메꾸는 방식으로 **팀만의 방향성을 확립**했습니다. 서로의 부족함을 탓하기보다 함께 해결하는 팀 문화를 조성하여 논의를 생산적으로 이끌었습니다.

</details>

<details>
<summary><b>3. 🌱 배운 점 및 향후 계획</b></summary>
<br>

* **배운 핵심 지식:**
    * **전체 시스템 구조 이해:** 프론트부터 백엔드, 데이터베이스, 배포(Docker, AWS)에 이르는 **서비스의 전체 개발 흐름**을 온전히 경험하고 이해했습니다.
    * **자기 주도적 학습의 중요성:** 스스로가 구현한 기능에 대해 명확히 설명할 수 있도록 **이론과 실습을 연결**하는 학습 태도가 중요하다는 점을 배웠습니다.
    * **협업의 가치:** 다양한 배경을 가진 팀원들과 공감하고 소통하며, 하나의 목표를 향해 나아가는 협업의 힘을 경험했습니다.
* **향후 서비스 발전 방향 또는 개선할 부분:**
    * **점진적 개선 및 확장:** 처음부터 거대한 규모를 목표로 하기보다, 현재의 기능을 기반으로 기술적인 검토와 수정을 꾸준히 진행하며 **차근차근 서비스 단계를 발전**시켜 나갈 계획입니다.
    * **고도화된 UX/UI 적용:** 사용자 피드백을 반영하여 직관적 사용성을 더욱 강화하고, 사용자 경험(UX) 중심의 설계 고도화를 추진할 것입니다.

</details>
